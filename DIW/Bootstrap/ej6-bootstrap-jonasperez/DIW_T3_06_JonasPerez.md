### TEORÍA

- **Roles ARIA**: Los roles ARIA (Accessible Rich Internet Applications) son atributos que se añaden al código HTML para decirle a las tecnologías asistivas (como los lectores de pantalla para personas con discapacidad visual) qué es y qué hace un elemento.

- **Contraste**: El contraste en Bootstrap se refiere a la legibilidad: asegurar que la diferencia de color entre el texto y el fondo sea lo suficientemente alta para que cualquier persona pueda leerlo, cumpliendo con las normas internacionales de accesibilidad (WCAG).

- **Tamaño de toque movil**: El tamaño de toque (o tap target size) en Bootstrap se refiere al área mínima que debe tener un elemento interactivo (botones, enlaces, inputs) para que un usuario pueda pulsarlo fácilmente con el dedo en un móvil sin equivocarse.

- **Formularios accesibles**: conectar cada campo con su `<label>` mediante el atributo id, permitiendo que los lectores de pantalla los identifiquen. Es fundamental usar aria-describedby para vincular textos de ayuda y asegurar que los errores no se indiquen solo con colores, sino también con texto.

- **Jerarquía visual**: La jerarquía visual es el orden en el que el ojo humano percibe la información. En Bootstrap, esto se logra organizando los elementos para que el usuario sepa qué es lo más importante (como un botón de compra) antes de leer los detalles secundarios.

- **Patrones UI comunes**: Los patrones de interfaz de usuario (UI) son soluciones estándar a problemas comunes de diseño. En Bootstrap, estos patrones ya vienen predefinidos para que no tengas que "reinventar la rueda" y el usuario se sienta familiarizado con tu web.
